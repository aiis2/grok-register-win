# 注册并发与凭据存储设计

日期：2026-07-19

## 目标

在保持现有单并发行为兼容的前提下，为面板注册任务增加 1–10 个固定并发槽。每个并发槽由独立 CLI 子进程承载，并独占一个浏览器实例与资料目录；同一槽内继续复用浏览器处理自己的账号分片。

同时把所有账号敏感产物集中到一个可配置的凭据根目录，支持在面板中修改目录并手动执行安全迁移。迁移范围包括 SSO 账号文件、临时邮箱凭据以及 CPA JSON、索引和失败记录。

## 非目标

- 不在单个 Python 进程内共享浏览器对象或邮箱状态。
- 不实现跨主机分布式注册或动态代理池调度。
- 不改变现有账号、CPA 和 Sub2 导出格式。
- 不覆盖迁移目标中已有且内容不同的文件。
- 不允许注册任务运行期间切换或迁移凭据目录。

## 并发架构

### 固定并发槽进程池

面板根据目标账号数和 `register_concurrency` 计算有效并发数：

```text
effective_concurrency = min(register_count, clamp(register_concurrency, 1, 10))
```

目标账号按连续区间均衡分配给并发槽。例如 10 个账号、3 并发会分为 4、3、3 个账号。每个槽拥有：

- 唯一 `worker_id`；
- 独立 CLI 子进程；
- 独立 Chromium/Camoufox 浏览器实例；
- 独立浏览器资料目录；
- 独立 SSO 和邮箱凭据输出文件；
- 独立轮次标记、超时状态和重启计数。

槽内账号串行执行并复用同一个浏览器。一个槽的账号超时只终止该槽拥有的 CLI 进程树，然后从该槽尚未完成的账号继续启动新进程；其他槽不受影响。

### 进程归属与停止

面板维护 `worker_id -> Popen` 的进程注册表。停止任务时对注册表快照中的每个进程执行精确 PID 树终止，不按进程名、窗口标题或浏览器可执行文件名扫描系统。

CLI 继续负责自己创建的浏览器根 PID 和子进程。每个 CLI 退出时只关闭自己的浏览器。并发槽之间不共享任何浏览器全局变量。

### 进度与日志

现有 `@@GROK_ROUND_START` 和 `@@GROK_ROUND_RESULT` 标记增加可选 `worker=<id>` 字段，但仍不包含邮箱、密码、SSO 或 JWT。面板按全局账号序号去重结果，并在状态接口中返回：

- 配置并发数与有效并发数；
- 活跃 worker 数；
- 每个 worker 的当前轮次、PID、状态和已完成数量；
- 汇总成功、失败和剩余数量。

## 凭据存储架构

### 目录模型

新增配置键 `credentials_dir`，默认值为 `data/credentials`。相对路径以应用根目录为基准解析，也支持可写的绝对路径。解析后的目录不能是文件系统根目录或应用根目录本身。

目录结构固定为：

```text
<credentials_dir>/
├── sso/     # accounts_*.txt
├── mail/    # 临时邮箱地址与 JWT
└── cpa/     # xai-*.json、index.json、failed.jsonl
```

集中路径解析放在新的 `lib/credential_store.py` 中，CLI、面板、下载、删除和 CPA 转换都通过同一接口获取路径，不再直接拼接仓库根目录或 `data/cpa`。

配置保存时：位于应用根目录内的路径保存为相对路径，外部路径保存为绝对路径，兼顾便携性和用户自定义。

### 并发安全输出

每个 worker 使用包含任务时间、worker ID、进程 ID 和随机短标识的文件名，避免同一秒启动时发生碰撞：

```text
sso/accounts_<timestamp>_w<worker>_<pid>_<id>.txt
mail/mail_credentials_<timestamp>_w<worker>_<pid>_<id>.txt
```

不同 worker 不写同一个 SSO 或邮箱凭据文件。CPA 转换仍由面板的单一队列汇总，单账号 CPA 文件名保持现有格式。

## 手动迁移

### 来源

迁移器识别两类来源：

1. 当前配置的旧凭据根目录；
2. 首次升级时的历史位置：仓库根目录 `accounts_*.txt`、`mail_credentials.txt` 和 `data/cpa`。

### 安全移动事务

用户在 UI 输入新目录并点击“迁移并切换”后：

1. 校验任务未运行、目标路径合法且可写；
2. 枚举来源文件并生成迁移计划；
3. 逐文件复制到目标临时文件；
4. 计算并比较 SHA-256；
5. 原子重命名临时文件为最终文件；
6. 同名同内容视为已迁移；同名不同内容自动追加时间戳和序号；
7. 全部复制校验成功后，原子写入新的 `credentials_dir` 配置；
8. 删除已验证的源文件，并清理空目录；
9. 返回迁移数量、跳过数量、冲突重命名数量和删除警告。

任何文件在配置切换前失败时，保留全部源文件且不修改配置。配置切换后的源删除失败只产生警告，因为已验证的新副本可以继续使用。

### UI

面板增加“凭据存储”卡片，包含：

- 当前配置值与解析后的绝对路径；
- 可写状态、SSO/Mail/CPA 文件数和总大小；
- 新目录输入框；
- “保存新路径”按钮：仅允许空目录或不存在目录，防止无意遗留旧凭据；
- “迁移并切换”按钮：执行完整安全移动；
- 迁移结果与警告提示。

注册任务运行时两个按钮禁用，后端接口也返回冲突状态，不能只依赖前端限制。

## 兼容性

- 缺少 `register_concurrency` 时按 1 处理。
- 缺少 `credentials_dir` 时使用 `data/credentials`，但在用户手动迁移前仍能发现历史账号文件用于下载和迁移提示。
- 下载和账号列表在迁移完成前兼容读取历史位置；迁移后以新目录为准并去重。
- 现有环境变量和 CLI 单批处理方式继续工作。
- Chromium 和 Camoufox 都能按槽隔离；Chromium 会显示每槽一个有头窗口。

## 错误处理与安全

- 并发值严格限制为 1–10，非法值在 UI 与 API 都被拒绝。
- 并发启动部分失败时，已启动槽继续完成；失败槽的账号记入明确失败状态或按受控重启策略继续。
- 迁移日志只记录路径、计数和文件名，不输出账号内容、SSO、密码或 JWT。
- 配置更新使用临时文件加 `os.replace`，避免中途写坏 `config.json`。
- 迁移目标不覆盖不同内容文件。
- 所有进程清理只作用于面板记录的 PID 树。

## 测试与验收

### 自动测试

- 并发值解析、边界与账号分片；
- 1、2、10 并发槽的进程启动和汇总；
- 每槽独立 worker ID、输出路径和环境变量；
- 单槽超时/重启不影响其他槽；
- 停止任务精确终止所有已登记进程；
- 凭据路径相对/绝对解析和安全限制；
- 并发文件名唯一性；
- 历史位置发现；
- 迁移成功、同内容跳过、异内容重命名、校验失败回滚和删除警告；
- UI 配置与迁移 API 在运行中拒绝操作；
- 下载、删除和 CPA 转换从新目录工作。

### 真实验证

- 面板桌面和移动端显示新增并发与凭据卡片，无控制台错误；
- 使用临时目录执行一次历史凭据完整迁移，核对 SHA-256、旧文件删除和下载结果；
- 启动至少 2 个 Chromium 并发槽，确认两个不同根 PID/Profile；
- 每槽连续执行多轮生命周期重置，确认槽内 PID 复用且进程数不持续增长；
- 停止后两个槽的所属 PID 全部退出，用户原有 Chrome/Edge 进程保持不变。
