# Panel V2 现代化重设计方案

## 目标

在不影响注册、邮箱、凭据、浏览器控制、下载和账号管理既有能力的前提下，提供一套可平滑迁移的现代面板：

1. 默认入口升级为清晰、紧凑、响应式的 V2 界面；经典界面长期保留并可一键回退。
2. 支持 `system`、`light`、`dark` 三种主题，刷新后保持偏好且首屏不闪烁。
3. 账号列表支持服务端搜索、过滤、排序和分页，不再把全部账号或敏感字段加载到浏览器。
4. 日志通过 SSE 实时增量刷新，并保留现有状态轮询作为兼容回退。
5. 所有既有操作在 V2 中达到功能等价，且局部故障不会拖垮整页。

## 现状与约束

当前面板由 `panel/app.py` 中的大段内联 HTML、CSS 和 JavaScript 构成。一次加载会渲染全部账号批次和大量配置控件；账号 API 也会读取完整记录。现有页面功能较完整，因此直接在原模板上重排会放大回归风险。

本轮遵守以下边界：

- 保留经典页面的 HTML、JavaScript 和既有 API 语义。
- 不引入 npm、前端框架、外部 CDN 或新的构建步骤。
- V2 使用 Flask、原生 HTML/CSS/JavaScript 和本地系统字体。
- 账号密码、SSO、access token、refresh token、邮箱管理密码等不得进入 V2 DOM、浏览器存储、日志或错误响应。
- 所有危险操作继续要求明确确认；所有写操作继续复用现有登录、互斥锁和后端校验。

## 方案选择

采用“独立 V2 模板和静态资源，共用后端契约”的迁移方式：

- 新增 `panel/templates/index_v2.html`。
- 新增 `panel/static/panel-v2.css` 和 `panel/static/panel-v2.js`。
- `/` 在功能等价验证完成后默认渲染 V2。
- `/?ui=legacy` 始终强制渲染经典页面。
- 迁移期间允许 `/?ui=modern` 显式访问 V2。
- V2 与经典界面仅通过后端 API 共享数据，不复用彼此 DOM 节点或页面状态。

没有选择直接改造原内联模板，因为它会把样式升级与既有行为强耦合；也没有选择 React/Vue SPA，因为当前项目无需承担额外构建与发布复杂度。

## 信息架构与页面布局

### 全局框架

桌面端采用吸顶顶栏、横向分区导航和最大宽度 `1440px` 的内容容器；不增加永久侧栏。窄屏下导航可横向滚动，所有内容转为单列。页面使用中性灰表面、单一蓝色强调色，成功、警告、错误颜色只用于表达状态。

顶栏持续显示任务状态，并提供主题切换、经典界面入口和关键快捷操作。V2 使用 hash 保存页面内深链：

- `#overview`
- `#register`
- `#accounts`
- `#mail`
- `#credentials`
- `#logs`

### 概览

展示任务状态、成功/失败数量、当前并发、账号和凭据指标、最近错误，以及开始注册、查看账号、查看日志等快捷操作。概览只汇总必要信息，不复制完整配置表单。

### 注册

保留轮数量、1–10 并发、浏览器模式、代理和全部启动/停止能力。Worker 以自适应矩阵呈现，每个 Worker 可独立查看、隐藏或恢复有头浏览器。停止注册等高影响操作使用可访问确认对话框。

### 账号

账号记录与“批次文件”分为两个视图：

- 账号记录支持搜索、来源过滤、状态过滤、排序、页码和每页数量。
- 批次文件保留现有预览、下载、选择和删除行为。
- 导入、导出和归档入口与列表状态联动，但不把凭据内容放入行数据。

### 邮箱

默认展示当前服务商和最常用连接配置；SMTP、Direct MX、发件测试等高级项折叠到“高级设置”。连接测试与端到端收件测试保留独立状态和脱敏错误详情。

### 凭据

集中展示当前凭据目录、容量和 CPA 转换状态，并保留批量导入、回填、下载、手动迁移及“与账号一起迁移”的全部能力。

### 日志

提供实时增量日志、暂停显示、自动滚动、级别过滤、关键字搜索和仅清空本地视图。暂停不会中断服务端任务或丢弃接收游标。

## 账号查询 API

新增只供 V2 使用的安全接口：

```http
GET /api/v2/accounts?page=1&page_size=25&q=email&source=batch&status=all&sort=newest
```

支持：

- `page`: 从 1 开始的页码。
- `page_size`: `25`、`50` 或 `100`。
- `q`: 邮箱模糊搜索。
- `source`: 账号来源或批次。
- `status`: `all`、`ready`、`pending`。
- `sort`: `newest`、`oldest`、`email`。

响应只包含规范化邮箱、来源、CPA 状态和必要时间元数据，以及分页总数。密码、SSO 和 OAuth 凭据永不返回。服务端以账号文件的文件名、大小和修改时间构建轻量快照缓存；文件变化、账号导入、删除、凭据迁移和注册结果写入后使缓存失效。排序必须稳定，非法分页和过滤参数返回明确的 400 错误。

前端对搜索输入使用约 `250ms` 防抖。切换搜索、过滤或页码时取消旧请求，并通过请求代次避免慢响应覆盖新状态。

## 实时日志协议

保留 `/api/job/status` 中现有日志字段，同时新增：

```http
GET /api/logs/stream?after=<sequence>
```

后端为日志分配进程内单调递增序号并维护有界缓冲区。SSE 事件使用序号作为 `id`，支持查询参数和 `Last-Event-ID` 断点续传，定期发送 heartbeat，并复用面板登录保护及日志脱敏逻辑。

V2 优先使用 `EventSource`；网络中断后从最后序号恢复，且不重复追加。若浏览器不支持 SSE、持续连接失败或服务端明确不可用，则回退到每 2 秒请求现有任务状态。日志连接故障只影响日志区域，不影响注册或其他配置。

## 前端状态与主题

V2 使用一个小型原生 JavaScript store 管理分区数据。首屏并行加载任务状态、浏览器配置、邮箱配置和凭据状态；账号数据只在进入账号分区时加载。

`localStorage` 仅保存：

- `system | light | dark` 主题偏好；
- 最近访问分区；
- 账号每页数量；
- 日志暂停、自动滚动和过滤等显示偏好。

任何密码、令牌、SSO 或邮箱管理凭据均不得写入浏览器存储。模板在样式加载前执行最小主题预绘制脚本，避免明暗主题闪屏；系统主题模式会响应操作系统变化。

字体仅使用本地字体栈：界面文字优先 `Segoe UI Variable`、`Microsoft YaHei UI`，日志和等宽指标优先 `Cascadia Mono`。动画尊重 `prefers-reduced-motion`。

## 加载、错误与可访问性

- 每个分区拥有独立 loading、empty、error 和 retry 状态。
- 请求进行时只锁定相关控件，不阻塞整页导航或任务状态刷新。
- 后端错误通过统一脱敏后显示，并保留可理解的阶段和重试建议。
- 确认操作使用焦点受控的对话框，支持 Escape、键盘导航和正确 ARIA 属性。
- 所有交互元素有可见焦点；状态不仅依赖颜色表达。
- 即使 V2 JavaScript 启动失败，服务端渲染的“切换到经典界面”入口仍可使用。

## 渐进式发布

1. 先实现安全账号分页接口、快照缓存和 SSE，经典页面完全不变。
2. 通过 `/?ui=modern` 加入 V2 外壳，逐区迁移并验证功能等价。
3. 自动化和真实浏览器验证通过后，最后一个切换提交才让 `/` 默认使用 V2。
4. `/?ui=legacy` 长期保留，发生回归时无需回滚后端即可恢复操作。
5. 更新 README，说明两套入口、主题、账号分页和日志行为，并记录明确的回退方法。

## 验证门槛

### 自动化测试

- `/`、`/?ui=modern`、`/?ui=legacy` 的渲染边界和本地静态资源。
- 账号分页参数、上限、搜索、过滤、稳定排序和缓存失效。
- API 响应与 V2 模板不含密码、SSO、access token 或 refresh token。
- 注册写入、账号导入/删除和凭据迁移后的账号快照一致性。
- SSE 登录保护、序号、断点续传、缓冲区滚动、heartbeat 和轮询兼容。
- 所有既有注册、邮箱、凭据、浏览器和下载测试继续通过。

### 浏览器验收

- 在 `1440`、`1280`、`768` 和 `390` 宽度下无页面级水平溢出。
- `system`、`light`、`dark` 切换、持久化和首屏无明显闪烁。
- 账号搜索、过滤、排序、换页、每页数量及空状态正确。
- 注册开始/停止、Worker 浏览器显隐、邮箱连接/收件测试、凭据导入/迁移和下载均可用。
- 日志实时到达，暂停、搜索、过滤与重连无重复。
- 键盘导航、焦点、对话框、ARIA 和减少动效模式可用。
- V2 与经典页面均无 JavaScript 控制台错误。

只有上述门槛全部通过后，才允许将 V2 设为默认入口。
