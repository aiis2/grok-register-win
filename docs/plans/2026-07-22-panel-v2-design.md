# Grok Register Win 稳定启动与 Panel V2 设计

## 目标

本轮改造同时解决三个问题：

1. Windows 启动器必须可靠启动面板，且 `python -m panel.app`、直接执行
   `panel/app.py` 两种入口保持一致。
2. 现有凭据工作区支持批量导入已有 SSO 并重新生成 OAuth 凭据；下载和删除只作用于
   当前账号集合，不携带已删除账号留下的凭据。
3. 新增简洁的 Panel V2，支持明暗主题与实时日志，同时完整保留现有页面作为回退入口。

## 约束与非目标

- 不改变现有注册、邮箱、凭据迁移、浏览器显隐和下载 API 的既有语义。
- 不删除 Legacy UI；V2 出现异常时可以立即切回旧页面。
- 不把 SSO、access token、refresh token、邮箱密码或管理凭据写入日志或前端错误。
- 不引入新的前端构建链，继续使用 Flask、原生 HTML/CSS/JavaScript。
- 不扩大浏览器、账号文件或凭据目录的清理范围。

## 1. 启动入口

`launcher.py` 以模块方式启动面板：

```text
python -m panel.app
```

工作目录固定为项目根目录。`panel/app.py` 在导入项目内包之前同时建立最小的项目根目录
bootstrap，使用户直接执行脚本时也能找到 `lib.*`。两条入口共享同一 `BASE_DIR`、配置和
日志目录。

启动测试分别验证模块入口和脚本入口能完成导入，不占用正式端口，也不启动长期后台任务。

## 2. 凭据工作区批量导入

### 数据流

1. API 接收 TXT 或 JSON 文件，限制大小并在内存中解析。
2. 使用现有 `parse_uploaded_records()` 规范化记录并按 SSO 去重。
3. 在当前凭据目录下创建 staging 目录，先写入新的账号文件并完成可读性校验。
4. 获取活动互斥锁，确认注册任务、邮箱收件测试、凭据迁移和 CPA 转换均未占用工作区。
5. 将旧的 SSO/CPA 文件移动到带时间戳的 archive 批次，再以原子替换方式激活 staging。
6. 为新批次逐条加入 CPA 转换队列，并返回批次 ID、解析数和入队数；响应不返回原始 SSO。

解析或 staging 失败时不触碰当前工作区。切换阶段失败时按移动清单回滚。归档位于可配置凭据
根目录内，并随“迁移并切换”一起迁移。

### 并发边界

- 注册任务运行中拒绝导入。
- 邮箱测试、凭据迁移或另一个导入正在运行时返回 HTTP 409。
- 切换工作区前清空尚未执行的旧批次转换任务；正在执行的转换必须先结束，不能与目录切换并发。
- 每个批次使用唯一 ID，状态响应只暴露计数、阶段、来源文件名和脱敏错误。

## 3. 当前账号范围的凭据导出

面板当前账号由规范化邮箱集合和 SSO SHA-256 指纹集合定义。CPA 文件满足任一条件才属于当前
账号集合：

- 文件内的 SSO 指纹匹配；
- 文件内的规范化邮箱匹配。

CPA ZIP、Sub2 ZIP/JSON、状态计数均使用同一筛选函数。删除账号文件后，孤立 CPA 不直接永久
删除，而是移动到当前凭据目录的 archive 批次，并同步清理失效索引记录。无法解析的 CPA 文件
不会进入下载包，也会以可恢复方式归档。

## 4. Panel V2

### 平滑迁移

- `panel_ui=modern` 渲染 V2；`?ui=legacy` 强制渲染旧页面。
- Legacy HTML、JavaScript 和 API 保持原样。
- V2 使用独立模板与静态 CSS/JavaScript，不复用易产生耦合的 DOM 节点。
- 两个页面只共享后端 API 契约；API 测试是迁移边界。

正式验证通过后默认使用 V2，同时在 V2 页脚提供“切换到经典界面”的恢复入口。查询参数只影响
当前浏览器会话，不写入全局配置。

### 信息架构

V2 使用五个区域：

1. 概览：任务状态、成功/失败、并发槽和凭据计数。
2. 注册：数量、并发、浏览器模式、代理节点及 Worker 卡片。
3. 邮箱：邮箱源配置和端到端收件测试。
4. 凭据：目录迁移、账号文件、批量导入与下载。
5. 日志：实时流、自动滚动、暂停显示和清空本地视图。

移动端使用单列布局；桌面端使用窄侧栏和内容区。危险操作保持二次确认，异步操作均显示忙碌、
成功和失败状态。

### 明暗主题

CSS 使用语义变量定义背景、卡片、文字、边框、强调色和状态色。首次进入读取
`prefers-color-scheme`，之后使用 `localStorage` 保存 `light`、`dark` 或 `system`。主题切换只改变
视觉变量，不重新加载页面或中断任务。

## 5. 实时日志

后端为每条日志分配进程内单调递增序号，并保留现有字符串队列供 `/api/job/status` 兼容使用。
新增只读 SSE 接口：

```text
GET /api/logs/stream?after=<sequence>
```

事件包含 `id` 和单条日志文本。服务器定期发送 heartbeat；浏览器通过 `Last-Event-ID` 或查询参数
断点续传。V2 优先使用 `EventSource`，连接失败时退回现有两秒轮询。重连只补发缓冲区内尚未收到
的事件，不重复整段日志。

SSE 接口复用面板登录校验，响应禁用缓存，不包含文件系统日志或敏感配置。

## 6. 错误处理与安全

- 所有导入错误通过统一脱敏器处理，原始上传内容不进入 traceback 响应。
- 文件名只保留安全 basename；archive 和 staging 路径必须验证位于当前凭据根目录。
- 下载筛选以内容身份为准，不信任文件名。
- SSE 客户端数量设置本地面板所需的合理上限；断开的生成器及时退出。
- Legacy 和 V2 的写操作继续使用相同的登录、活动锁和输入校验。

## 7. 验证与发布门槛

### 自动化测试

- 两种面板启动入口及 launcher 命令。
- TXT/JSON 解析、去重、大小限制、繁忙状态、staging 失败和切换回滚。
- 可配置凭据目录中的 archive、迁移和账号范围导出。
- 删除后的孤立 CPA 可恢复归档与索引一致性。
- SSE 顺序、断点续传、heartbeat、登录保护和轮询兼容。
- V2 模板包含所有关键功能入口，Legacy 路由仍可访问。

### 真实验证

- 使用独立端口启动 `start.bat` 对应的 launcher 流程并通过 `/health`。
- 在 Chromium 中验证 light、dark、system 三种主题持久化。
- 触发模拟日志，确认无需等待两秒即可到达页面；中断连接后自动补发且不重复。
- 检查桌面与窄屏布局、控制台错误、网络失败提示和 Legacy 回退。
- 对注册、邮箱测试、凭据迁移、下载和 Worker 浏览器控制执行关键路径回归。
