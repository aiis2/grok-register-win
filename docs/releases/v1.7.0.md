# v1.7.0 · 面板启动与隐藏有头浏览器加固

本版本由 `aiis2` 整理并发布，重点解决 Windows 启动器无法加载项目模块、隐藏有头
Chromium 启动时仍可能闪烁或抢占焦点，以及账号凭据导入导出范围不够明确的问题。

## 主要变更

- 修复面板启动失败：启动器改用 `python -m panel.app`，避免直接执行
  `panel/app.py` 时项目根目录不在 `sys.path`，导致
  `ModuleNotFoundError: No module named 'lib'`。
- 保留直跑兼容：即使用户或旧批处理仍直接执行 `python panel/app.py`，面板也会先将
  项目根目录加入模块搜索路径，继续正确加载 `lib`。
- 增加无副作用启动探针：自动化测试可以只验证完整导入链，不启动后台转换线程或
  Flask 服务，防止同类入口回归。
- 加固隐藏有头浏览器启动：Windows `hidden` 模式在创建原生窗口前加入屏幕外初始
  坐标、`STARTUPINFO/SW_HIDE` 和 CDP 后台无焦点参数；浏览器仍是完整有头 Chromium，
  不加入任何 `--headless` 参数。
- 改进 Win32 隐藏控制：仅对当前 worker 捕获且 PID/HWND 同时匹配的窗口执行
  `SWP_NOACTIVATE` 隐藏、任务栏样式调整和最终可见性校验，自动启动与重启路径不会
  主动调用前台激活 API。
- 修正窗口恢复判断：不再把 `ShowWindowAsync` 的“调用前可见状态”返回值误判为
  操作失败，而是检查最终窗口状态。
- 修复隐藏窗口零尺寸：部分 Chrome 版本在 `SW_HIDE` 启动后返回零尺寸矩形；现在会
  在屏幕外预置可恢复的 `1280×800` 窗口，用户点击“显示浏览器”时将同一 PID/HWND
  移回可见工作区，再次隐藏不会重建浏览器或丢失当前页面。
- 收紧凭据导出范围：CPA、Sub2 等导出只包含当前账号集合拥有的有效凭据；无主、损坏
  或已删除账号对应文件转入独立归档，并同步整理索引。
- 增加批量凭据安全导入：上传内容先完成大小限制、格式解析、去重和暂存校验，再归档
  当前工作区并原子激活新批次；激活失败会回滚已移动文件。
- 注册任务、CPA 转换、凭据迁移或另一导入正在运行时拒绝工作区切换，避免并发写入造成
  账号与派生凭据不一致。

## 兼容性与安全边界

- Windows 默认隐藏模式仍是有头浏览器；不会自动降级成无头模式。
- 隐藏引导失败时只清理本次捕获的精确进程树，并沿用最小化有头兼容回退。
- 不按进程名或模糊窗口标题操作用户自己的 Chrome/Edge。
- 原有 `python panel/app.py`、新版 `python -m panel.app` 和 Windows 启动批处理均受支持。
- 配置文件、Release 和测试中不包含邮箱、SMTP、代理、账号或其他真实凭据。

## 验证

- 合并后的完整测试：`278 passed`。
- `python panel/app.py` 无副作用启动探针输出 `PANEL_STARTUP_OK`。
- `python -m panel.app` 独立端口启动后，`/health` 返回 HTTP `200` 和 `ok=true`。
- Windows 独立 Chrome/Profile 探针在 721 ms 启动阶段高频采样 249 次，本次浏览器成为
  前台窗口次数为 0；隐藏、非激活显示和再次隐藏均复用同一 PID/HWND。
- 探针结束后对应浏览器进程和临时 Profile 均已清理。

浏览器前台控制最终仍受 Windows 版本、Chromium 版本和桌面安全策略影响。本版本通过
启动前屏幕外放置、CDP 后台创建和 Win32 无激活隐藏降低闪烁与失焦风险，但不承诺所有
第三方窗口管理工具环境下绝对零焦点变化。
