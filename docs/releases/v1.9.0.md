# v1.9.0 · 隐藏有头窗口加固与 SSO 全量换票

本版本由 `aiis2` 整理并发布，重点修复隐藏有头 Chromium 仍可能出现在 Windows
任务栏的问题，并为新版与经典面板补齐已有 Web SSO 的批量 OAuth/CPA 刷新能力。

## 主要变更

- 修复真实主窗口识别：按本次启动的精确 PID 优先选择
  `Chrome_WidgetWin_1`，不再把先枚举到的内部 `Chrome_WidgetWin_0` 当作浏览器主窗口。
- 加固隐藏稳定性：首次隐藏后继续执行有界扫描，捕获延迟出现或被替换的主窗口；
  隐藏完成后窗口不可见、带 `WS_EX_TOOLWINDOW`、不带 `WS_EX_APPWINDOW`，不保留
  可见主窗口或任务栏入口。
- 保持完整有头能力：隐藏模式不添加 `--headless`，静默启动和非激活显示不会主动
  抢占当前前台窗口；手动显示与再次隐藏继续复用同一 worker 的 PID/HWND。
- 新增“刷新全部 SSO 换票”：使用账号池中已有 Web SSO 重新换取 OAuth/CPA，
  不生成新的 Web SSO；单次处理上限支持 `1–10000`。
- 新版“凭据 → CPA 转换”加入确认模态框、队列计数和页面内错误提示；经典界面
  `?ui=legacy` 同步提供等效按钮，升级后仍可平滑回退。
- 刷新任务复用现有串行 CPA worker：已完成项允许重新入队，正在处理的相同 SSO
  会跳过，避免重复换票。
- CPA 成功结果通过同目录临时文件原子替换；换票或写入失败保留旧 CPA，不删除、
  不截断当前可用凭据。
- 刷新 API 与注册、凭据导入和凭据迁移使用一致的非阻塞互斥顺序；忙碌时返回明确
  错误，响应只包含总数、入队数、跳过数和公共 CPA 状态。

## 兼容性与安全边界

- Chromium 的 `hidden`、`minimized`、`visible` 三种模式及 Camoufox 入口保持兼容。
- 浏览器控制仍只作用于任务登记的精确 PID/HWND，不按进程名扫描或控制用户自己的
  Chrome/Edge。
- 原有自动 CPA、缺失 CPA 补转、下载、Sub2 导出和凭据迁移接口继续保留。
- 全量刷新不回显 Web SSO、密码、Access Token、Refresh Token 或邮箱管理密钥。
- 配置、测试与 Release 不包含真实邮箱、代理、服务器地址或账号凭据。

## 验证

- 完整自动化测试：`374 passed`。
- `node --check panel/static/panel-v2.js` 通过。
- `python -m compileall -q grok_register_ttk.py launcher.py panel lib tests` 通过。
- Windows Chrome 150 原生探针确认选中 `Chrome_WidgetWin_1`；隐藏后主窗口不可见，
  `TOOLWINDOW=true`、`APPWINDOW=false`，目标 PID 下可见主窗口为 0。
- 原生探针确认启动前后及非激活显示时前台 HWND 不变；显示、再次隐藏复用相同
  PID/HWND，命令行无 headless 参数，User-Agent 不含 `HeadlessChrome`。
- 真实浏览器验证新版按钮、确认取消、空工作区错误回显与经典界面回退；取消确认时
  未发送刷新请求。
- 新版在 1440px 与 390px 视口均无页面级横向溢出，正常页面控制台错误为 0。

升级时保留现有 `config.json`、`data/credentials/` 与 `data/logs/`，替换程序文件后重新
运行 `start.bat` 即可。若新版界面在特定环境出现兼容问题，可访问
`http://127.0.0.1:8787/?ui=legacy` 使用经典界面。
