# v1.14.0 · CPA 拒绝诊断与安全熔断

本版本由 `aiis2` 整理并发布，针对“注册成功但 CPA OAuth 连续返回 `Access denied`”的场景补充结构化诊断、批量止损和并发竞态保护。明确拒绝的账号继续进入禁用池，尚未尝试的账号则保持原状，避免在服务端拒绝期间继续消耗整个账号池。

## 主要变更

- 新增结构化 OAuth 授权拒绝类型，记录拒绝阶段、HTTP 状态、OAuth 错误和响应传输方式。
- 分别识别 RSC consent action、303 callback 和 token exchange 返回的 `access_denied`，不再把回调拒绝误报为“缺少 code”。
- token 端错误只保留经过截断和换行清理的安全字段，不把完整响应正文写入失败信息。
- 连续 3 个不同账号被服务端明确拒绝时，自动 CPA 与批量重新授权进入熔断暂停。
- 已实际收到拒绝的账号仍写入 `disabled` 账号池，并继续从 SSO、CPA、Sub2 和聚合导出中排除。
- 尚未执行的账号只从当前 CPA 队列中跳过，不会因为批量熔断被误标记为禁用。
- worker 在真正执行 OAuth 前再次检查熔断状态，封住熔断置位与并发入队之间的竞态窗口。
- 面板状态区展示熔断原因，便于区分服务端授权拒绝与网络、代理、回调或响应解析故障。

## 行为说明

- 单个或少量账号的明确拒绝仍按账号级失败处理，不会立即暂停整个 CPA 管线。
- 达到连续拒绝阈值后，当前批次状态变为 `paused`，未尝试账号计入跳过数量并保留原凭据。
- 自动注册产生的后续 CPA 入队会被拒绝，但注册得到的 Web SSO 仍保存在凭据目录，可在服务恢复后主动重新授权。
- 用户主动点击“重新生成账号授权”会开启新的预检批次；如果服务端仍连续拒绝，批次会再次安全暂停。
- 429、临时 5xx、网络超时、Cloudflare 页面和协议解析变化继续使用各自的错误分类，不会被误判为账号授权拒绝。

## 验证

- 完整自动化测试：`544 passed`。
- 回归覆盖 RSC action 拒绝、303 callback 拒绝、token exchange 拒绝、批次熔断、自动队列清空和 worker 并发竞态。
- Python `compileall`、面板启动导入检查与 `git diff --check` 通过。
- 发布差异未包含真实邮箱、密码、Web SSO、Access Token、Refresh Token、邮箱管理配置或测试服务器信息。

## 升级说明

1. 停止旧面板并保留本地 `config.json`、`data/credentials/` 与 `data/logs/`。
2. 更新项目文件后重新运行 `start.bat`。
3. 如果页面显示 OAuth 连续拒绝熔断，请先保留未尝试账号，不要反复批量提交。
4. 确认授权条件恢复后，再通过“重新生成账号授权”执行新的单账号预检和批量换票。

## 安全边界

- `Access denied` 是授权服务返回的明确拒绝结果；本项目只负责诊断、隔离和止损，不伪造身份或绕过服务端授权决策。
- 禁用账号登记仍属于本地敏感凭据，不应提交、上传或公开。
