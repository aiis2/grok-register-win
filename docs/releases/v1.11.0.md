# v1.11.0 · CPA OAuth 并行性能与注册稳定性

本版本由 `aiis2` 整理并发布，将 CPA OAuth 网络兑换升级为有界并行流水线，同时加固并发注册在 Turnstile、完成页跳转、SSO Cookie 与任务汇总阶段的稳定性。现有凭据格式、导出协议和经典界面保持兼容。

## 主要变更

- CPA OAuth 网络阶段默认使用 2 个 worker，并可在新版面板配置为 `1–4`；每个 worker 使用独立 Session、PKCE、state 与 nonce。
- OAuth 结果统一进入单提交器，串行完成 CPA JSON 原子替换、`index.json` 更新、失败记录和最终状态提交，避免多个网络 worker 互相覆盖索引。
- 新版“凭据 → CPA 转换”显示运行并发、活跃 worker、待兑换和等待提交数量；修改并发度后会明确提示重启面板生效。
- 同一 SSO 在排队、网络兑换和等待提交阶段都不会重复入队；凭据导入或迁移会等待整个流水线空闲，并通过工作区 generation 阻止旧结果写入新目录。
- 仅对 429、短暂 5xx 和网络超时执行最多两次有界退避；SSO 失效、OAuth 协议错误和 Cloudflare 403 不盲目重试。
- 换票或提交失败时保留原有 CPA 文件；索引使用原子写入，失败记录由单提交器顺序追加。
- Chromium 注册继续保留 GPU 能力以完成 Turnstile，并减少多 worker 同时验证时的无效重试和页面抖动。
- 注册状态机识别已经完成的跳转，等待规范的 Grok SSO Cookie 后再保存凭据，并按真实已完成轮次汇总并发任务结果。

## 性能结果

隔离性能探针使用 24 个任务、固定 80ms 合成 OAuth 延迟和真实文件/索引提交：

| OAuth worker | 总耗时 | 吞吐 | 相对单 worker | 文件 / 索引 |
| ---: | ---: | ---: | ---: | ---: |
| 1 | 2.418 秒 | 9.92/秒 | 1.00× | 24/24 · 24/24 |
| 2 | 1.219 秒 | 19.69/秒 | 1.98× | 24/24 · 24/24 |
| 4 | 0.647 秒 | 37.07/秒 | 3.74× | 24/24 · 24/24 |

真实 OAuth 对照使用两组各 4 个既有且此前转换成功的 Web SSO，只记录匿名化计数与耗时：

| OAuth worker | 成功 / 失败 | 总耗时 | 吞吐 |
| ---: | ---: | ---: | ---: |
| 1 | 4 / 0 | 16.378 秒 | 14.65/分钟 |
| 2 | 4 / 0 | 7.180 秒 | 33.43/分钟 |

默认 2 worker 的端到端总耗时提升为 `2.28×`；两组索引均为 4/4，未出现 429、403、超时或协议错误。所有测试结束后，请求队列、提交队列、inflight、worker 和提交线程均归零。

## 兼容性与升级

- 现有 `config.json` 无需迁移；缺少 `cpa_oauth_concurrency` 时默认使用 2。
- `CPA_CONCURRENCY=1..4` 可作为部署环境覆盖；设为 1 可恢复串行网络兑换，同时继续使用安全的单提交器。
- 修改 OAuth 并发度后需要重启面板，正在执行的任务不会被动态缩放。
- 现代面板、经典界面、邮箱配置、注册与日志、账号分页、凭据迁移、SSO 刷新及 CPA/Sub2API 导出协议保持不变。
- 升级时保留本地 `config.json`、`data/credentials/` 与 `data/logs/`，替换程序文件后重新运行 `start.bat`。

## 安全边界

- 状态 API、页面、日志、性能结果与本说明均不包含 Web SSO、Access Token、Refresh Token、密码或邮箱管理密钥。
- 并发 worker 不共享 Session、Cookie、PKCE 参数或 token。
- 凭据迁移和批次导入仍会在 CPA 请求、兑换或提交任一阶段繁忙时拒绝执行，避免跨工作区写入。
