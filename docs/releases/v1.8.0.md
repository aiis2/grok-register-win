# v1.8.0 · 现代面板与安全实时控制台

本版本由 `aiis2` 整理并发布，在保留原有注册、邮箱、凭据和浏览器控制能力的基础上，
将本地管理面板升级为默认的现代化控制台，并保留经典界面作为可逆回退入口。

## 主要变更

- 新增现代面板：按概览、注册、账号、邮箱、凭据和日志六个分区重新组织工作流，
  支持跟随系统、浅色和深色主题。
- 平滑迁移默认入口：访问 `/` 或 `/?ui=modern` 使用新版，访问
  `/?ui=legacy` 可随时回到经典界面；切换界面不会修改配置、账号或凭据。
- 完整迁移注册控制：支持 `1–10000` 轮、`1–10` 并发、浏览器模式选择、任务停止，
  以及每个 worker 的浏览器显示与隐藏控制。
- 增加账号目录 API：支持服务端分页、关键词搜索、来源与 CPA 状态过滤及排序；
  页面可选择 `25/50/100` 条每页，并保留批次导入、预览、下载与删除能力。
- 加固账号数据边界：账号列表仅返回邮箱、来源、时间和 CPA 就绪状态，不返回密码、
  SSO、OAuth Token 或邮箱管理密钥。
- 完整迁移邮箱配置：支持现有全部邮箱适配器、密钥安全保存、连接测试、发件能力检查，
  以及端到端收件验证；已保存密钥只显示状态，不回显到页面或浏览器存储。
- 完整迁移凭据仓库：支持查看目录与容量摘要、修改保存位置、手动迁移、CPA 队列状态
  和补转操作。
- 新增实时日志通道：使用带序号的 SSE 增量推送，支持断线续传、去重、暂停显示、
  自动滚动、级别和关键词过滤；连续连接失败时自动回退到任务状态轮询。
- 新版前端保持零构建依赖：不需要 Node.js、npm 或外部 CDN，继续由 Flask 直接提供
  本地 HTML、CSS 和 JavaScript。
- 修复实时日志追平状态和窄屏账号表格的可访问性定位问题，避免连接状态长时间停留在
  “连接中”，并消除 390px 视口的页面级横向溢出。

## 兼容性与安全边界

- 原有注册、邮箱、凭据迁移、CPA 转换、浏览器显隐和任务停止接口继续保留。
- 经典界面固定保留在 `/?ui=legacy`，无需迁移配置即可回退。
- 主题、日志显示和账号分页等前端偏好只保存非敏感数据；密码、Token 和管理密钥不会
  写入 localStorage。
- 日志在浏览器中最多保留 2000 条；页面“清空”只清除本地显示，不删除服务端日志。
- 配置、测试和 Release 不包含邮箱、SMTP、代理、账号或其他真实凭据。

## 验证

- 合并前及合并后的完整测试均通过；最终测试为 `355 passed`。
- `node --check panel/static/panel-v2.js` 通过。
- `python -m compileall -q grok_register_ttk.py launcher.py panel lib tests` 通过。
- 真实浏览器验证默认 `/` 加载六个新版分区，`/?ui=legacy` 正常回退经典界面。
- 1440px 桌面和 390px 移动视口均无页面级横向溢出，最终导航控制台错误为 0。
- SSE 实时连接、断线恢复、暂停缓冲、过滤、本地清空和轮询回退均完成浏览器验证。

升级时保留现有 `config.json`、`data/credentials/` 与 `data/logs/`，替换程序文件后重新
运行 `start.bat` 即可。若新版界面在特定环境出现兼容问题，可先使用
`http://127.0.0.1:8787/?ui=legacy` 继续操作。
