# v1.10.0 · 注册日志合并与大日志性能优化

本版本由 `aiis2` 整理并发布，重点解决大量注册后打开实时日志可能长时间卡死的问题，并将注册操作和日志观察合并到同一个工作区。

## 主要变更

- 现代面板将原来的“注册”和“日志”主导航合并为“注册与日志”：任务参数、执行状态和 Worker 控制位于上方，实时日志控制台位于下方。
- 保留旧 `#logs` 深链兼容；历史书签或概览中的“查看日志”会进入合并页面并定位到日志区域。
- 浏览器内存仍保留最近 `2000` 条日志，但默认只绘制最近 `300` 条匹配记录，避免日志规模直接等于 DOM 节点规模。
- 新增“加载更早”和“回到最新”；加载更早每次增加 `300` 条，并保持当前阅读位置。
- SSE 历史回放和实时事件改为约 `100ms` 的批量刷新，不再为每一条事件重新过滤并重建全部日志节点。
- 搜索输入使用短防抖；级别筛选、暂停/恢复、自动滚动、重新连接、轮询回退和清空本地显示继续保留。
- 切换到其他主分区时仍可接收日志，但不执行不可见的日志 DOM 重建；重新进入后进行一次有界同步。

## 根因与性能结果

旧实现会在每条 SSE 事件到达时执行一次整表过滤、DOM 全量替换和滚动布局。2000 条历史事件回放因此形成接近二次复杂度的页面更新。

Playwright 在隔离的 Chromium 中验证同一组 2000 条日志：

- 优化前出现约 49.3 秒的单次 Long Task，并创建 2000 个日志行节点；
- 优化后从点击“注册与日志”到 2000 条完成接收为 `726ms`；
- 默认日志节点稳定为 300，最长 Long Task 为 `74ms`；
- 额外 500 条突发日志在 `468ms` 内显示到最新记录，期间 Long Task 为 0，主题切换保持响应；
- 搜索得到 500 条匹配时默认显示 300 条，加载更早可显示全部 500 条，回到最新恢复 300 条；
- 暂停期间继续接收但冻结 DOM，恢复后一次同步到最新日志。

这些数据用于验证性能退化已经消除，不承诺所有硬件上的绝对耗时完全相同。

## 兼容性与安全边界

- `/api/logs/stream`、`/api/job/status`、事件序号、断点续传和轮询回退协议不变。
- 经典界面 `?ui=legacy` 不受导航和渲染调整影响。
- 日志不会新增浏览器持久化或远程上传；“清空本地显示”仍不会删除服务端任务日志。
- 密码、Web SSO、Access Token、Refresh Token 和邮箱管理密钥不会进入日志性能测试、截图或 Release 内容。
- 账号分页、邮箱收件测试、凭据迁移、SSO 全量刷新、CPA 和 Sub2API 导出接口保持兼容。
