# v1.5.0 · Generic Email Receive Testing

本版本由 `aiis2` 整理并发布，汇总公开仓库建立以来完成的主要功能变更。

## 主要变更

- 原生集成 Cloudflare Temp Email：增加独立邮箱适配器、管理员配置 UI、地址 JWT 收件、连接探测与临时地址清理，同时保留旧配置兼容。
- 加固有头 Chromium 生命周期：每个 worker 使用独立调试端口和临时 Profile，同一槽内复用健康浏览器；超时、断连和停止只清理本任务登记的精确进程树。
- 支持 `1–10` 个固定并发注册槽：每槽独享 CLI 进程与有头浏览器，任务分片、进度、PID 和成功/失败结果可在面板查看。
- 增加可配置凭据仓库：SSO、邮箱 JWT 与 CPA 统一按子目录保存；UI 支持修改路径和整库迁移，并以复制、SHA-256 校验、原子切换、来源清理完成可验证迁移。
- 增加通用端到端邮箱收件验证：所有已接入收件适配器可通过 Freemail 原生 API、SMTP Relay 或用户显式启用的 Direct MX 发送单封测试邮件，并复用 Grok 验证码读取逻辑完成精确验证。
- 增加异步测试状态机和 UI 模态框：展示创建、快照、发送、等待、比对和清理阶段，支持取消、超时与安全错误反馈；测试和注册任务互斥，避免资源冲突。
- 重建项目 README，补充 Windows 快速开始、并发资源模型、邮箱配置、凭据迁移与排障说明；完善自动化测试和 Windows CI 验证。

## 安全与默认行为

- Freemail 可从 `MAIL_WEB_URL`、`ADMIN_NAME`、`ADMIN_PASSWORD` 读取运行时配置，页面显式配置优先；真实凭据不会写入仓库。
- Freemail 与 SMTP 密码不通过配置 API 回显，空密码提交保留已有值。
- 测试状态和错误不会返回验证码、完整测试邮箱、密码、Bearer Token 或 Cookie。
- Direct MX 默认关闭，并仅向本次生成邮箱域名的 DNS MX 主机投递；25 端口封锁和反垃圾策略可能导致失败。

## 验证范围

发布前执行完整 pytest、Python 编译、浏览器桌面/窄屏流程，以及 Freemail 登录、能力探测、测试邮箱创建与清理验证。原生发件成功路径要求 Freemail 部署侧另行配置 Resend、SendFlare 或 Cyberpersons 发件渠道；未配置时 UI 会返回明确的 `sending` 阶段错误。实际注册成功率仍取决于代理出口、目标平台风控、邮箱域名信誉和第三方服务状态。
